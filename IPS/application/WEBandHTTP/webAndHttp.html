WEB / HTTP

WEB

WEB은 World Wide Web의 약자이다. 웹은 인터넷 프로토콜 기반의 네트워크에서 도메인 이름이나 IP 주소, 루트 경로만으로 이루어진 일반 URL을 통하여 보이는 웹 페이지 (Web Page)들의 의미 있는 묶음이다. 대한민국에서 흔히 말하는 홈페이지는 엄밀히 말해 웹사이트를 지칭한다. 최초의 웹사이트는 팀 버너스리가 1990년에 CERN에서 만든 info.cern.ch이다. 웹사이트는 인터넷이나 랜과 같은 네트워크를 통해 접속할 수 있는, 적어도 하나의 웹 서버 상에서 호스팅된다. 웹 페이지는 HTML/XHTML의 형식으로 표현되지만 일반적으로 순수 문자열로 쓰여진 문서이다. 웹 페이지는 HTTP를 통하여 접속되며 가끔씩은 HTTPS를 통한 암호화를 사용하여 웹 페이지 콘텐츠를 이용한 사람들에게 보안과 개인 정보 보호를 제공한다. 사용자가 이용하는 웹 브라우저는 HTML 마크업 명령을 모니터에 표시하는 그대로 페이지 내용을 표현한다. 공식적으로 접속할 수 있는 모든 웹사이트는 총체적으로 월드 와이드 웹을 이루고 있다.

WEB APPLICATION DEVELOP LANGUAGE(PROGRAMMING LANGUAGE)

웹 개발 언어는 크게 Client의 Front-End 언어와 Server의 Back-End 언어로 구분할 수 있다. Client-Server 모델에서 Client가 일반적으로 Front-End로, Server가 Back-End로 구분된다.

PROGRAMMING LANGAUGE: CLIENT(FRONT-END)

HTML(HyperText Markup Language)는 웹페이지를 만들기 위한 마크업 언어이다. HTML은 꺽쇠 괄호(<, >)를 사용해 제목, 단락, 목록과 같은 본문을 구분하는 HTML 요소를 이용하여 웹페이지를 구조화된 문서로 작성한다. HTML은 웹 브라우저와 같은 HTML 처리 장치의 행동에 영향을 주는 자바스크립트와 본문과 그 밖의 항목의 외관과 배치를 정의하는 CSS 같은 스크립트를 포함하거나 불러올 수 있다. HTML과 CSS 표준의 공동 책임자인 W3C는 명확하고 표상적인 마크업을 위하여 CSS의 사용을 권장한다.
CSS(Cascading Style Sheet)는 마크업 언어가 실제 표기되는 방법을 기술하는 언어로 W3C의 표준이며 레이아웃과 스타일을 정의할 때 사용한다. HTML이 웹사이트의 골격을 구성한다면 CSS는 웹사이트의 외관을 꾸미는 역할을 담당한다고 할 수 있다. 즉, HTML 구조는 그대로 두고 CSS 파일만 변경해도 전혀 다른 웹사이트처럼 꾸밀 수 있다.
JavaScript는 객체 기반의 스크립트 프로그래밍 언어로 웹 브라우저 내에서 주로 사용하며, 웹페이지를 조작할 뿐만 아니라 다른 응용 프로그램의 내장 객체에도 접근할 수 있는 기능을 가지고 있어 클라이언트와 서버간 상호작용과 관련된 일을 수행할 수 있다.

PROGRAMMING LANGUAGE: SERVER(BACK-END)

JSP(Java Server Pages)는 HTML 내에 Java 코드를 삽입하여 웹서버에서 동적으로 웹페이지를 생성하여 웹브라우저에게 리턴해주는 서버 사이드 스크립트 언어이다. Client에서 서비스가 요청(Request)되면 JSP의 실행을 요구하고, JSP는 웹 어플리케이션 서버의 서블렛 컨테이너에서 서블릿 원시 코드로 변환된다. 이후 서블릿 원시코드는 컴파일된 수 실행되어 결과를 HTML 형태로 Client에게 리턴해준다.
ASP(Active Server Pages)는 마이크로소프트사(이하 MS사)에서 동적으로 웹페이지들을 생성하기위해 개발한 서버 사이트 스크립트 엔진이다.
PHP(Hypertext Preprocessor)는 동적 웹페이지를 만들기 위해 설계되어 PHP 처리 기능을 가진 웹 서버에서 코드를 인식하여 사용자가 원하는 웹페이지를 생성하는 명령형 프로그래밍 언어이다.

HTTP 기본 도식도

HTTP(HyperText Transfer Protocol)

HTTP란 텍스트 기반의 통신 규약으로 인터넷에서 데이터를 주고받을 수 있는 프로토콜이다. 모든 프로그램과 운영체제, 소프트웨어가 이 규약에 맞춰져 개발되어 서로 정보를 교환할 수 있게 되었다.
웹페이지는 Object(개체)로 구성되어있다. Object는 HTML 파일, JPEG 이미지, Java Applet, 오디오 파일 등등 수많은 종류가 있다. 웹페이지는 여러 참조 객체를 포함하는 기본 HTML 골격(Skeleton)에서 시작한다. 각각의 개체는 URL이라 불리는 주소를 가진다.

HTTP

HTTP(HyperText Transfer Protocol)

HTTP란 W3(인터넷)상에서 데이터(문서와 같은 리소스)를 주고받을 수 있는 프로토콜로, 클라이언트와 서버 사이에 이루어지는 요청/응답(request/response) 프로토콜이다. HTTP는 웹에서 이루어지는 모든 데이터 교환의 기초로 클라이언트인 웹 브라우저는 HTTP를 통하여 서버로부터 웹페이지(HTML)나 그림과 같은 정보를 요청(request)하면, 서버는 이 요청에 응답(response)하여 필요한 정보를 해당 사용자에게 전달하게 된다.
이 정보는 모니터와 같은 출력 장치를 통해 사용자에게 나타난다. HTTP를 통해 전달되는 자료는 http:로 시작하는 URL(인터넷 주소)로 조회할 수 있다. HTTP는 클라이언트-서버 프로토콜로 수신자(보통 웹브라우저) 측에 의해 요청이 초기화되는 프로토콜을 의미한다.

HTTP connection

HTTP/1.0은 가장 기초적인 형태의 웹 프로토콜로 TCP Protocol 위의 HTTP를 이용한 HTTP Protocol에서 동작한다. 웹의 규모가 거대해지면서 클라이언트와 서버간 Request와 Response의 시간이 급증해버리는 Network Latency 이슈가 발생했고 이를 해결하기 위해 다양한 방법이 연구되었다.

non-persistent HTTP

HTTP 통신을 통해 여러 오브젝트를 요청 및 응답하는 경우, HTTP 1.0 초기에는 각각의 요청마다 TCP 연결을 3-Way Hanshake 방식으로 맺어야했다. 3-Way Hanshake란 아래 SYN, SYN-ACK, ACK 과정으로 이루어진다. 클라이언트와 서버 사이에 TCP 연결을 생성하기 위해 클라이언트가 서버에게 SYN 이라는 특별한 패킷을 전송한다. SYN 패킷을 받은 서버는 클라이언트로부터 TCP 연결 요청을 인지하고 SYN, ACK 패킷을 리턴한다(성공은 ACK, 실패는 NAK). 서버로부터 SYN, ACK 패킷을 리턴받은 클라이언트는 다시 ACK 패킷을 리턴함으로써 TCP 연결을 생성한다. 이후 HTTP 요청, 처리, 응답의 과정을 거친 뒤 TCP 연결을 종료한다. 새로운 HTTP 통신을 위한 TCP 연결 요청이 필요할 경우 위의 과정을 반복한다. 만약 10개의 오브젝트를 지닌 하나의 웹페이지를 요청하고 응답하기 위해서는 클라이언트와 서버는 10번의 3-Way Hanshake 과정을 수행해야했다. 웹 초창기에는 웹을 통해 전달하는 웹페이지의 오브젝트, 콘텐츠 수가 많이 않았기 때문에 다수의 TCP 연결 사용으로 인한 부담이 크지 않았지만 웹사이트의 콘텐츠에 이미지와 같은 멀티미디어 수가 증가하면서 1회성 TCP 연결로는 부족하게되었다.

persistent HTTP

이를 해결하기 위해 나온 기술이 Persistent Connection 기술이다. HTTP 1.1에서는 Persistent 기능을 이용하여 1개의 TCP 연결에 여러 오브젝트 요청이 가능하다. TCP 연결의 재사용으로 서버는 TCP 세션 처리 부하를 줄일 수 있었고, 클라이언트는 응답 속도가 개선되었다.

HTTP Format

클라이언트와 서버 사이의 소통은 평문(ASCII) 메시지로 이루어진다. 클라이언트는 서버로 요청메시지를 전달하며 서버는 응답메시지를 보낸다.

REQUEST MESSAGE

클라이언트가 서버에게 보내는 요청 메시지의 예시

GET / ~~~ / v1.0 HTTP / 1.1
Accept: application/json
Authorization: ~~~

RESPONSE MESSAGE

서버가 클라이언트에게 보내는 응답 메세지 예시

HTTP/1.1 200 OK
Date: ~~day, dd mm yy tt:tt:tt GMT
Conttent-Type: hext/html; charset=UTF-8
Content-Encoding: UTF-8
Content-Length: ~~
Last-Modified: ~~day, dd mm yy tt:tt:tt GMT
Server: Apache/n.n.n.n (UNIX)
ETag: "~~~"
Accept-Ranges: bytes
Connection: close

~~~~html code~~~~

데이터 전송 기술, TCP와 UDP

클라이언트와 서버는 HTTP 통신을 이용하여 서로 약속된 양식으로 데이터 정보를 주고받는다. HTTP 통신 뿐만 아니라 데이터를 전송하기위해 사용하는 전송 계층의 프로토콜이 필요한데 대표적으로 TCP와 UDP가 있다. 데이터는 네트워크 선로를 통해 패킷이란 단위로 전달되고 패킷은 각자의 길을 통해 목적지에 도달하는데 이 과정에서 패킷이 손실되거나 패킷의 순서가 뒤바뀌어 전달될 수 있다. 네트워크를 이용한 데이터 전송에서 중요한 요소는 Data integrity(신뢰성), Throughput(전송률), Timing(시간), Security(암호화, 보안)의 4가지 요소이다. TCP는 높은 신뢰성과 보안을 제공하고 UDP는 높은 전송률과 빠른 속도를 제공한다. TCP와 UDP는 Transport Layer 단원에서 좀 더 자세히 공부한다.
