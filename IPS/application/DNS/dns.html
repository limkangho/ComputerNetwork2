DNS

Domain

Domain이란 사람이 이해하고 기억하기 어려운 IP 주소에 이름을 부여한 것이다. IP(Internet Address)란 인터넷에 연결된 장치들이 각각의 식별된 주소를 의미하며 숫자로 이루어져있다. 컴퓨터는 IP 주소를 이용하여 인터넷 상에서 다른 장치의 주소를 찾아 정보를 요청하거나 요청에 대한 응답을 리턴하거나 웹사이트에 접속할 수 있다. Domain은 컴퓨터 장치의 이름과 최상위 도메인, 그리고 그 하위 도메인으로 구성된다. 예를 들어 'naver.com' 에는 'naver'라는 장치의 이름과 'com'이라는 최상위 도메인을 말한다. Domain을 사용하더라도 결국 IP 주소를 알 수 있어야 목적 장치와 연결이 가능하다.

Domain Name Space

Domain Name Space는 Domain Name System이 관리하는 계층 구조를 의미한다. 인터넷 도메인 체계에서 최상위 계층으로 인터넷 도메인의 시작점인 Root 계층이 존재하고 하위로 .com, .edu, .gov, .mil과 같은 하위 도메인들이 존재한다. PC 저장 장치에서 사용하는 디렉토리(폴더) 구조와 유사하여 각 레벨의 도메인은 하위 도메인을 관리한다.

Name Server

컴퓨터 장치는 인터넷 네트워크 상에서 목적 장치의 IP 주소를 찾기 위해 문자열로 표현된 도메인 이름을 숫자로 표현된 IP 주소로 변환할 필요가 있다. 그리고 이렇게 도메인 이름을 IP 주소로 변환할 때에 Domain Name Space에 대한 정보가 필요한데 이때 이 Domain Name Space의 트리 구조에 대한 정보를 가지고 있는 서버를 Name Server라 한다. 전 세계에 13개의 Root DNS 서버가 구축되어있다.

Resolver

Resolver는 웹 브라우저와 같은 DNS 클라이언트의 요청을 Name Server로 전달하고 Name Server로부터 Domain name과 IP address를 받아 클라이언트에게 제공하는 기능을 말한다. 사용자가 특정 웹사이트에 방문하기 위해 Domain을 입력한다면 Resolver는 하나의 Name Server에 DNS 요청을 전달하고 해당 서버에 정보가 없다면 같은 계층의 다른 Name Server에 요청을 보내고 마지막으로 하위 계층의 Name Server에 순차적으로 요청을 보낸다. Resolver가 질의(Query)를 보내면 Name Server에서는 응답(Reply)을 리턴한다.

DDoS Attack

DDoS(Distributed Denial of Service) Attack이란 특정 서버(컴퓨터) 혹은 장치를 대상으로 수많은 데이터 트래픽을 발생시켜 장애를 일으키는 해킹 기법이다. 대상 서버에 비정상적으로 많은 트래픽을 발생시켜 과도한 트래픽 처리, 프로세스 소모, 입출력 등을 이용해 서버가 수용할 수 없는 양의 데이터 처리를 요구한다.
